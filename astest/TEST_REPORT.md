## 一、已确认可用的功能

### 1.1 类型系统

| 类型 | 中文关键字 | 可用性 | 说明 |
|------|-----------|--------|------|
| 整数 (i32) | `整数` / `整型` | 完全可用 | 默认整数类型 |
| 小数 (f64) | `小数` | 完全可用 | 默认浮点类型 |
| 布尔 (i1) | `布尔` | 完全可用 | 字面量 `真`/`假`，支持比较/逻辑运算 |
| 字符串 (i8*) | `字符串` | 完全可用 | 支持中英文引号 |
| 无 (void) | `无` | 完全可用 | 用于函数返回类型 |
| 数组 | `数组` | 完全可用 | 支持所有类型元素（整数/小数/布尔/字符串等） |
| 浮点 (f32) | `浮点` | **不可用** | 无f32字面量，所有小数被解析为f64 |
| 字符 (i8) | `字符` | **不可用** | 无字符字面量，整数字面量为i32 |

### 1.2 变量声明与赋值

- `定义 变量：类型 名称` —— 完整声明（不可变）
- `定义 可变 变量：类型 名称 = 初始值` —— 可变声明
- `设 名称 = 表达式` —— 简化声明（可变，类型推断）
- `令 名称 为 表达式` —— 等价于 `设`
- `=` 和 `为` 均可作为赋值运算符
- 类型推断在 `设`/`令` 中正常工作

### 1.3 运算符

| 类别 | 运算符 | 状态 |
|------|--------|------|
| 算术 | `+` `-` `*` `/` `%` | 通过（仅支持同类型） |
| 比较 | `==` `!=` `<` `<=` `>` `>=` | 通过 |
| 逻辑 | `&&` `\|\|` `!` `且` `或` | 通过 |
| 一元 | `-` `!` | 通过 |
| 优先级 | 括号 `()` `（）` | 通过 |
| 字符串比较 | `==` `!=` | **编译崩溃** |

### 1.4 控制流

- `判断` / `若` / `否则` —— if/else-if/else
- `当前` / `取` / `此外` —— switch/case/default
- `循环计数 i < N` —— counted for-loop
- `循环迭代 i < start..end` —— range for-loop
- `循环条件 expr` —— while-loop
- 嵌套控制流、循环中判断均正常

### 1.5 函数

- `定义 方法 名称（参数：类型，...）返回 类型：...。。`
- `@声明 入口` —— 入口点标记
- `@声明 外部` —— 外部函数声明（FFI）
- 递归调用正常（阶乘、斐波那契验证通过）
- `执行 target：args` —— void 函数调用
- `target（args）` —— 有返回值函数调用
- `取值 target [->类型] [：args]` —— 另一种调用形式

### 1.6 数组

- 字面量: `【1，2，3】` 或 `[1, 2, 3]` (中英文方括号)
- 多类型支持: 从小数 `【1.5，2.7】`、布尔 `【真，假】`、字符串 `["a", "b"]` 等类型推断
- 预定义: `定义 变量：数组 arr[10]` (默认整数元素)
- 类型推断定义: `定义 变量：数组 arr = 【3.14, 2.71】` (推断为小数数组)
- 索引: `arr[0]` 或 `arr【1】`
- 元素赋值: `设 arr[0] = 10` 或 `arr[1] 为 20`
- 编译期常量下标越界检测

### 1.7 结构体

- `定义结构名称：字段：类型，...。。`
- `构造名称：字段：值，...。。` 创建实例
- `实例->字段` 字段访问
- `设 实例->字段 = 值` / `令 实例->字段 = 值` 字段赋值
- 字段在 f-string 中可通过 `{实例->字段}` 插值

### 1.8 对象

- `定义对象名称：结构：...构造方法...公共成员/私有成员...。。`
- `创建对象名（参数）` 实例化
- `实例->方法（参数）` 方法调用
- `当前` 在对象方法中表示 self
- 公共/私有成员访问控制（私有方法外部不可调用）
- 对象字段默认私有（外部不可直接访问）

### 1.9 字符串

- 英文引号 `"..."` 和中文引号 `"..."` 均支持
- f-string: `f"文本 {变量}"` 变量插值
- f-string: `{结构体->字段}` 字段插值
- `{{` 和 `}}` 输出花括号

### 1.10 模块系统

- `#模块 名称` 声明模块
- `引用 模块：路径` 导入模块
- `引用 模块：路径 为 别名` 别名导入
- 函数内可以作用域导入 (`引用` 作为语句)

## 二、缺陷与限制

### 2.1 严重缺陷 (Bugs)

#### B1. 字符串比较导致编译器崩溃
```
设 b = "hello" == "hello"
```
触发: `panicked at codegen.rs:1600:22: Found PointerValue but expected IntValue`

**影响**: 字符串的 `==` / `!=` 比较在代码生成阶段 panic，无法使用。

#### B2. 无运行时数组越界检查
```
定义 变量：数组 arr[3]
设 idx = 5
设 v = arr[idx]  // 编译通过，运行时读取栈上垃圾数据
```

**影响**: 变量下标访问不做运行时检查，可能导致内存安全问题。

#### B3. 除零无错误处理
```
设 z = 10 / 0  // 编译通过，运行返回 0（LLVM undef 行为）
```

**影响**: 整数除零无保护，产生不确定结果。

### 2.2 类型系统缺陷

#### D1. `浮点` (f32) 类型不可用
- 所有小数被解析为 `小数` (f64)，无法创建 f32 值
- `定义 变量：浮点 x = 1.0` 报类型不匹配错误
- **建议**: 增加 f32 字面量语法，如 `1.0f` / `1.0浮`，或实现隐式类型转换

#### D2. `字符` (i8) 类型不可用
- 无字符字面量语法，无法创建 Char 值
- `定义 变量：字符 c = 65` 报类型不匹配（65 是 Int，不是 Char）
- **建议**: 增加字符字面量语法，如 `'A'` / `'A'`

#### D3. `布尔` 类型无字面量 —— ✅ 已修复
- 已增加 `真`/`假` 关键字作为布尔字面量
- 支持布尔变量声明、赋值、函数返回、数组元素

#### D4. 数组仅支持整数元素 —— ✅ 已修复
- 数组字面量现在从元素推断元素类型，支持整数、小数、布尔、字符串等所有类型
- `定义 变量：数组 arr = 【3.14, 2.71】` 现在可以正常创建小数数组
- 数组元素类型一致性仍被严格检查（所有元素必须同类型）

### 2.3 格式化字符串限制

#### F1. 不支持表达式
`f"结果: {a + b}"` —— 报"未定义的格式化变量"

#### F2. 不支持数组索引
`f"值: {arr[0]}"` —— 报"未定义的格式化变量"

#### F3. 不支持布尔值格式化 —— ✅ 已修复
`f"flag: {boolVar}"` —— 布尔值格式化为 `%d` (0 或 1)。

### 2.4 语法解析限制

#### P1. 无法在 `定义 变量：` 中指定自定义类型
`定义 变量：坐标 p = ...` 把 `坐标` 当变量名而非类型名，导致解析错误。只能通过 `设` 进行类型推断。

**建议**: 解析器在 `变量：` 后若当前 token 不是内置类型关键字，应尝试将其识别为结构/对象类型名。

#### P2. 关键字冲突导致标识符不可用
关键字作为前缀的标识符会被拆分词法分析。例如:
- `计数器` 被拆为 `计数` (Count关键字) + `器` (标识符)
- `计数值` 同理

**建议**: 词法分析器对中文标识符使用最长匹配原则（目前对多字符关键字如 `字符串` 应该已支持，但对 `计数` 这类2字关键字未正确处理后续字符）。

#### P3. 同模块内带模块前缀调用失败
在模块内调用 `模块名-函数名(...)` 会解析为 `模块名-模块名-函数名`，导致找不到函数。只能使用不加前缀的直接名称。

### 2.5 功能缺失

| 缺失功能 | 说明 | 优先级 |
|---------|------|--------|
| break/continue | 循环中无法提前跳出 | 高 |
| 类型转换 | Int↔Double 等转换 | 高 |
| 混合类型算术 | Int + Double 报错 | 中 |
| 字符串拼接 | 无 `+` 拼接 | 中 |
| 多维数组 | 不支持嵌套数组 | 中 |
| 默认参数值 | 函数参数无默认值 | 低 |
| 函数重载 | 同名函数不支持 | 低 |
| 三元运算符 | 无 `? :` 等价物 | 低 |
| 常量/枚举 | 无 `const`/`enum` | 低 |
| 块注释 | 仅支持 `//` | 低 |
| 标准输入 | `获取输入` 声明但可能存在参数问题 | 中 |
| 结构体方法 | 结构体只能通过独立函数操作 | 中 |
| for-each 循环 | 只能通过下标遍历数组 | 中 |
| 返回值忽略 | 有返回值函数不能用 `执行` 调用 | 低 |

### 2.6 其他设计注意事项

1. **`设` 声明的变量总是可变的** —— 与 `定义 变量` 默认不可变的行为不一致，可能导致混淆。
2. **对象字段外部不可访问** —— 即使读取也需要通过 getter 方法，没有 `public` 字段机制。
3. **`若` 不能独立使用** —— `若` 只能在 `判断` 之后作为 elseif，不能作为独立的 if。
4. **`否则` 必须在 `判断` 或 `若` 之后** —— 不能单独使用。
5. **循环变量不可变** —— 循环计数/迭代中的循环变量不能被重新赋值。
6. **选择语句的 case 值类型必须与目标变量类型一致** —— 类型严格匹配，不允许隐式转换。

## 三、编译错误信息质量评估

### 优点
- 错误信息包含中文描述，对中文用户友好
- 多数错误有 `帮助:` 提示给出修正建议
- 不可变变量赋值错误会显示声明位置和错误位置（类 Rust 风格）
- 类型不匹配错误会显示期望类型和实际类型

### 存在的问题
- 代码生成阶段的错误信息不够详细（如 Bool 格式化仅提示"暂不支持"）
- 编译器 panic (字符串比较) 时无友好错误信息
- 解析错误在遇到关键字冲突时，提示可能误导用户

## 五、结论

问源编程语言的核心语法功能已基本实现，编译器能够成功编译并运行涵盖变量、运算符、控制流、函数、数组、结构体、对象、字符串、模块等主要特性的程序。测试覆盖了 100+ 个用例，均通过编译和运行验证。

但存在以下关键问题需要优先修复:
1. **字符串比较导致编译器崩溃** (B1)
2. **无运行时数组越界检查** (B2)
3. **除零无保护** (B3)
4. **`浮点` 和 `字符` 类型不可用** (D1, D2)
5. **格式化字符串功能受限** (F1-F3)
6. **自定义类型无法在完整变量声明中使用** (P1)
7. **关键字前缀冲突导致中文标识符不可用** (P2)
